Опис CI/CD пайплайну для проєкту Shop Zara

Пайплайн реалізовано за допомогою платформи GitHub Actions та хмарного сервісу Render.com для автоматизації процесів збірки, тестування та розгортання контейнеризованого Django-застосунку.

Пайплайн складається з двох основних етапів (jobs):

1. Етап: Build & Test (Безперервна інтеграція - CI)
Тригер: Пуш коду в гілку main/master.
Процес: GitHub Actions піднімає ізольоване середовище (ubuntu-latest), переходить у директорію backend та збирає Docker-образ застосунку. Після успішної збірки система запускає контейнер і виконує команду автоматичного юніт-тестування (python manage.py test). Для забезпечення швидкості та ізоляції тестів на цьому етапі використовується локальна база даних SQLite.

2. Етап: Deploy (Безперервне розгортання - CD)
Умова: Виконується ТІЛЬКИ у випадку успішного проходження всіх тестів на попередньому етапі.
Процес: GitHub Actions використовує захищений секретний ключ (GitHub Secrets), щоб зробити curl-запит на Deploy Hook платформи Render. Отримавши сигнал, Render автоматично ініціює новий білд: викачує актуальний код з репозиторію, компілює Production-версію Docker-контейнера, підключається до хмарної бази даних PostgreSQL через змінні середовища (Environment Variables), застосовує міграції та запускає веб-сервер через Gunicorn. 

Безпека: Усі чутливі дані (паролі до бази даних, секретні ключі Django, Deploy Hook) зберігаються у зашифрованому вигляді у змінних середовища Render та GitHub Secrets і не потрапляють у систему контролю версій.
